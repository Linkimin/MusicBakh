# Чеклист релиза

Последовательность шагов для выпуска новой версии MusicBakh. Пройти от начала до конца до того, как ставить тег и публиковать установщик.

## 1. Подготовка кода

- [ ] Все изменения для версии слиты в `main`.
- [ ] `dotnet test` — все тесты зелёные.
- [ ] `dotnet build -c Release MusicLibrary/MusicLibrary.csproj` — собирается без предупреждений.
- [ ] Версия в [MusicLibrary.csproj](../MusicLibrary/MusicLibrary.csproj) обновлена (`<Version>`, `<AssemblyVersion>`, `<FileVersion>`).
- [ ] При необходимости — короткий CHANGELOG-абзац в commit-сообщении тега или отдельном `CHANGELOG.md`.

## 2. Ручной smoke на dev-машине

Гонится прямо `dotnet run --project MusicLibrary`. Проверить, что все кнопки работают:

- [ ] Импорт локального `.mp3` (через «＋ Добавить трек» → «Из файла»).
- [ ] Импорт по URL (через «＋ Добавить трек» → «По ссылке»).
- [ ] Play / Pause / Stop, перемотка ±10 с, переход к предыдущему/следующему.
- [ ] Активный seek-слайдер: клик по дорожке и перетаскивание.
- [ ] Слайдер громкости и кнопка mute.
- [ ] Переключение режима повтора: No → Current → Library.
- [ ] Удаление пользовательского трека: открывается кастомный `ConfirmationDialogWindow`, трек исчезает из библиотеки и из `userTracks.json`.
- [ ] Сохранение трека через «Сохранить» (`SaveFileDialog`).
- [ ] Двойной клик по карточке трека и по записи в истории — оба запускают воспроизведение.
- [ ] Фильтр по жанру.
- [ ] Горячие клавиши: `Space`, `←`/`→`, `Ctrl+←`/`Ctrl+→`, `M`, `R`.
- [ ] Перезапуск приложения: громкость, mute и режим повтора восстановились из `%LocalAppData%\MusicBakh\player-settings.json`.

## 3. Сборка установщика

- [ ] Inno Setup 6 установлен (`C:\Program Files (x86)\Inno Setup 6\ISCC.exe`).
- [ ] Запустить:

      ```powershell
      pwsh scripts/build-release.ps1 -Version X.Y.Z
      ```

- [ ] Скрипт прошёл без ошибок, в `release\` появился `MusicBakh-Setup-X.Y.Z.exe`.
- [ ] Размер установщика — в разумных рамках (ожидаем 60–80 МБ при single-file compressed).

## 4. Smoke на чистом окружении

Идеально — Windows Sandbox (`WindowsSandbox.exe`) или виртуальная машина / соседская Windows без .NET.

- [ ] Установка `MusicBakh-Setup-X.Y.Z.exe` (SmartScreen → «Подробнее → Выполнить»).
- [ ] Установка per-user в `%LocalAppData%\Programs\MusicBakh`, UAC не запрошен.
- [ ] В «Параметры → Приложения» виден `MusicBakh X.Y.Z`, издатель `Matvey Bakhmatov`.
- [ ] В меню «Пуск» появился ярлык `MusicBakh` с фирменной иконкой.
- [ ] Запуск — окно открывается с правильной иконкой в taskbar и заголовком `MusicBakh`.
- [ ] Эталонные 3 трека видны в библиотеке, обложки подтянулись.
- [ ] Импортирован сторонний `.mp3` — играет, метаданные подтягиваются.
- [ ] После закрытия в `%LocalAppData%\MusicLibrary\` и `%LocalAppData%\MusicBakh\` появились файлы данных.
- [ ] Повторный запуск — состояние сохранилось.
- [ ] Деинсталляция через «Параметры → Приложения»: `{app}` удалён, `%LocalAppData%\MusicBakh\` остался (пользовательские данные).
- [ ] Повторная установка — старая библиотека пользователя подхватилась.

## 5. Публикация

- [ ] Создать аннотированный тег: `git tag -a vX.Y.Z -m "Release X.Y.Z"`.
- [ ] Запушить тег: `git push origin vX.Y.Z`.
- [ ] Создать GitHub Release из тега, прикрепить `MusicBakh-Setup-X.Y.Z.exe`.

## Что не входит в текущий процесс (на будущее)

- **Code signing.** Сертификат не куплен — SmartScreen будет предупреждать. Когда сертификат появится: `signtool sign /tr http://timestamp.digicert.com /td sha256 /fd sha256 /a publish\win-x64\MusicBakh.exe` до запуска ISCC, и повторно тем же `signtool` на готовом `MusicBakh-Setup-*.exe`.
- **Авто-обновление.** Сейчас релиз ручной. При появлении регулярных версий можно посмотреть на Velopack/Squirrel.
- **GitHub Actions.** Сборка ручная — при росте частоты релизов имеет смысл вынести в CI.
- **ARM64.** Только x64.
