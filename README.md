# MusicBakh

Настольный музыкальный плеер и библиотека для Windows. Хранит локальную коллекцию треков, читает теги, подтягивает обложки и жанры, проигрывает аудио со всеми привычными элементами управления.

## Возможности

- **Импорт треков** — добавление в библиотеку из локального файла (mp3/wav) или по прямой ссылке (до 50 МБ).
- **Воспроизведение** — Play/Pause, перемотка ±10 секунд, переход к следующему/предыдущему треку, активный seek-слайдер.
- **Громкость и mute** — отдельный слайдер 0–100% и кнопка беззвучного режима.
- **Режимы повтора** — без повтора / повтор текущего трека / повтор всей видимой библиотеки.
- **Горячие клавиши** — `Space`: play/pause, `←`/`→`: перемотка ±10 с, `Ctrl+←`/`Ctrl+→`: предыдущий/следующий, `M`: mute, `R`: смена режима повтора.
- **Метаданные** — каскад «теги ID3 → очистка от мусорных суффиксов агрегаторов → MusicBrainz → имя файла».
- **Жанры на русском** — словарь из 50+ маппингов (`rock` → «Рок», `electronic` → «Электроника» и т. д.).
- **Обложки** — встроенная картинка ID3 → iTunes Search API (600×600) → процедурная заглушка с буквой исполнителя.
- **История воспроизведения** — последние 50 запусков, двойной клик переигрывает.
- **Фильтр по жанру** — выпадающий список, автоматически собирается из библиотеки.
- **Сохранение и удаление** — экспорт выбранного трека в файл, удаление пользовательских треков с подтверждением.
- **Persistence** — громкость, mute и режим повтора восстанавливаются между запусками.

## Требования

| Сценарий | Что нужно |
|---|---|
| Запуск из инсталлятора | Windows 10 1809 (build 17763) или Windows 11, x64, ~200 МБ свободного места |
| Запуск из исходников | дополнительно [.NET 10 SDK](https://dotnet.microsoft.com/) |

Инсталлятор поставляется как self-contained сборка — отдельно ставить .NET-рантайм пользователю не нужно.

## Установка

1. Скачать `MusicBakh-Setup-X.Y.Z.exe` из раздела Releases.
2. Запустить. Windows SmartScreen может предупредить о неподписанном исполняемом файле — «Подробнее → Выполнить в любом случае».
3. Установщик ставит приложение per-user в `%LocalAppData%\Programs\MusicBakh` (UAC не требуется).

Пользовательские данные хранятся в:

- `%LocalAppData%\MusicLibrary\` — импортированные треки (`Music\`), обложки (`Covers\`), индекс `userTracks.json`.
- `%LocalAppData%\MusicBakh\player-settings.json` — громкость, mute, режим повтора.

При деинсталляции каталог приложения удаляется полностью, пользовательские данные в `%LocalAppData%` сохраняются, чтобы переустановка восстановила библиотеку.

## Запуск из исходников

```powershell
git clone https://github.com/Linkimin/MusicBakh.git
cd MusicBakh
dotnet build
dotnet run --project MusicLibrary
```

Тесты:

```powershell
dotnet test
```

## Сборка релиза

```powershell
pwsh scripts/build-release.ps1 -Version 1.0.0
```

Скрипт публикует single-file self-contained сборку под win-x64 в `publish\win-x64\` и собирает установщик через Inno Setup в `release\MusicBakh-Setup-1.0.0.exe`. Подробности — в [docs/release-checklist.md](docs/release-checklist.md).

## Архитектура

WPF + лёгкий MVVM на .NET 10. Зависимости собираются вручную в [MainWindow.xaml.cs](MusicLibrary/MainWindow.xaml.cs:20), без IoC-контейнера. Полное описание слоёв и сервисов — [docs/architecture.md](docs/architecture.md).
